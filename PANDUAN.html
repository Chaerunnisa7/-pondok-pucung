🏛️ Panduan Website Kelurahan Pondok Pucung

Tutorial lengkap membuat website profil kelurahan menggunakan Next.js + deploy ke Vercel.
Cocok untuk pemula yang sudah mengenal HTML & CSS.

🛠️ 1. Persiapan — Install Dulu Ini

Sebelum mulai coding, kamu perlu menginstal beberapa software:

A Install Node.js

Node.js adalah "mesin" yang menjalankan JavaScript di komputermu (bukan di browser).

  1. Buka nodejs.org
  2. Download versi LTS (yang lebih stabil)
  3. Install seperti biasa (Next, Next, Finish)
  4. Verifikasi: buka Terminal/Command Prompt, ketik:
node --version
npm --version

Jika muncul angka versi (misal: v20.11.0), berarti berhasil! ✅

B Install VS Code

Text editor terbaik untuk web development. Download di code.visualstudio.com

Extensions yang disarankan (install di VS Code):

  • ES7+ React/Redux/React-Native snippets — shortcut kode React
  • Prettier — auto-format kode jadi rapi
  • Auto Rename Tag — rename tag HTML otomatis

📁 2. Struktur Folder Project

Ini adalah struktur folder final project kamu. Pelajari dulu sebelum mulai coding:

pondok-pucung/ ← Folder utama project ├── src/ ← Source code ada di sini │ ├── app/ ← Setiap folder = satu halaman web │ │ ├── layout.js ← Template utama (navbar+footer) │ │ ├── page.js ← Halaman Home (URL: /) │ │ ├── globals.css ← CSS global untuk semua halaman │ │ ├── profil/ │ │ │ └── page.js ← Halaman Profil (URL: /profil) │ │ ├── visi-misi/ │ │ │ └── page.js ← Halaman Visi Misi (URL: /visi-misi) │ │ ├── struktur/ │ │ │ └── page.js ← Halaman Struktur (URL: /struktur) │ │ ├── berita/ │ │ │ └── page.js ← Halaman Berita (URL: /berita) │ │ └── kontak/ │ │ └── page.js ← Halaman Kontak (URL: /kontak) │ ├── components/ ← Komponen yang dipakai ulang │ │ ├── Navbar.js ← Menu navigasi (muncul di semua hal.) │ │ ├── Footer.js ← Footer (muncul di semua halaman) │ │ └── PageHeader.js ← Header halaman dalam │ └── data/ │ └── data.js ← Semua data statis (berita, dll) ├── package.json ← Daftar library yang dipakai └── next.config.js ← Konfigurasi Next.js (mode static)
💡
Aturan paling penting Next.js App Router:
Setiap folder di dalam app/ yang punya file page.js = satu halaman website.
Contoh: folder profil/page.js → halaman dengan URL /profil

📦 4. Instalasi Project

1 Buka Terminal

Di Windows: tekan Win + R → ketik cmd → Enter
Di Mac: tekan Cmd + Space → ketik Terminal → Enter

2 Buat Project Next.js Baru
npx create-next-app@latest pondok-pucung

Akan muncul beberapa pertanyaan. Jawab seperti ini:

✔ Would you like to use TypeScript? → No
✔ Would you like to use ESLint? → Yes
✔ Would you like to use Tailwind CSS? → No  (kita pakai CSS biasa)
✔ Would you like to use `src/` directory? → Yes
✔ Would you like to use App Router? → Yes
✔ Would you like to customize the default import alias? → No
3 Masuk ke Folder Project
cd pondok-pucung
4 Jalankan Server Development
npm run dev

Buka browser, akses http://localhost:3000 — website kamu sudah berjalan! 🎉

🏗️ 5. Layout & Komponen Reusable

Layout adalah "bingkai" yang membungkus semua halaman. Navbar dan Footer ada di sini sehingga tidak perlu ditulis berulang di setiap halaman.

// src/app/layout.js — ini dijalankan PERTAMA sebelum halaman apapun
import './globals.css'      // Import CSS global
import Navbar from '@/components/Navbar'
import Footer from '@/components/Footer'

// Metadata = informasi untuk Google (SEO)
export const metadata = {
  title: 'Kelurahan Pondok Pucung',
  description: 'Website resmi kelurahan...',
}

export default function RootLayout({ children }) {
  return (
    <html lang="id">
      <body>
        <Navbar />         {/* Muncul di SEMUA halaman */}
        <main>{children}</main>  {/* Isi halaman berubah di sini */}
        <Footer />         {/* Muncul di SEMUA halaman */}
      </body>
    </html>
  )
}
🔵
Apa itu @/?
Tanda @/ adalah shortcut untuk folder src/.
Jadi @/components/Navbar = src/components/Navbar.js
Ini supaya tidak perlu menulis path panjang seperti ../../components/Navbar

Menggunakan Komponen PageHeader

// Import komponen yang sudah dibuat
import PageHeader from '@/components/PageHeader'

export default function ProfilPage() {
  return (
    <>  {/* <> = Fragment, pembungkus tanpa div ekstra */}

      <PageHeader
        judul="Profil Kelurahan"
        deskripsi="Mengenal lebih dekat Kelurahan Pondok Pucung"
      />
      
      {/* Isi konten halaman di bawahnya */}
      <section>
        <p>Konten halaman di sini...</p>
      </section>
    </>
  )
}

🗃️ 7. Mengelola Data Statis

Karena website ini tidak pakai database, semua data (berita, struktur, dll) disimpan di file JavaScript:

// src/data/data.js — edit file ini untuk update konten!

export const dataBerita = [
  {
    id: 1,
    judul: 'Gotong Royong RT 05',
    tanggal: '20 April 2025',
    kategori: 'Kegiatan',
    ringkasan: 'Deskripsi singkat berita...',
    emoji: '🧹',
  },
  // Tambahkan berita baru di sini dengan format yang sama
  {
    id: 2,
    judul: 'Judul Berita Baru',
    tanggal: '25 April 2025',
    kategori: 'Kesehatan',
    ringkasan: 'Ringkasan berita baru...',
    emoji: '🏥',
  },
]

Cara menggunakannya di halaman:

// Di halaman berita (src/app/berita/page.js)
import { dataBerita } from '@/data/data'  // Import data

export default function BeritaPage() {
  return (
    <div>
      {/* .map() = loop/perulangan, tampilkan tiap item */}
      {dataBerita.map((berita) => (
        <div key={berita.id}>  {/* key wajib ada saat loop */}
          <h3>{berita.judul}</h3>
          <p>{berita.tanggal}</p>
          <p>{berita.ringkasan}</p>
        </div>
      ))}
    </div>
  )
}

🎨 8. CSS & Styling

Kita menggunakan CSS biasa (bukan Tailwind) sehingga lebih mudah dipahami pemula.

CSS Variables — Cara Mendefinisikan Warna

/* Di globals.css */
:root {
  /* Definisikan warna utama sekali */
  --hijau-tua: #1a4731;
  --hijau-muda: #52b788;
  --emas: #c9a84c;
}

/* Gunakan di mana saja */
.navbar {
  background: var(--hijau-tua);  /* pakai variable */
}

.harga {
  color: var(--hijau-muda);
}

Cara Menulis Style Inline di JSX

// Di JSX, style inline pakai objek JavaScript (camelCase!)
// HTML biasa:   style="background-color: red; font-size: 14px"
// JSX:          style={{ backgroundColor: 'red', fontSize: '14px' }}

<div style={{
  backgroundColor: 'var(--hijau-tua)',
  padding: '24px',
  borderRadius: '16px',
}}>
  Konten
</div>
⚠️
Perbedaan HTML vs JSX:
  • HTML: class="nama" → JSX: className="nama"
  • HTML: for="id" → JSX: htmlFor="id"
  • HTML: <br> → JSX: <br /> (harus ditutup)

🔗 9. Navigasi Antar Halaman

Di Next.js, jangan gunakan tag <a> biasa untuk link internal. Gunakan komponen Link:

// Selalu import Link dari next/link
import Link from 'next/link'

// ❌ SALAH — ini akan reload seluruh halaman
<a href="/profil">Profil</a>

// ✅ BENAR — ini navigasi cepat tanpa reload
<Link href="/profil">Profil</Link>

// Link dengan className
<Link href="/profil" className="btn btn-primer">
  Lihat Profil
</Link>

// Link ke halaman luar (eksternal) — pakai <a> biasa
<a href="https://tangsel.go.id" target="_blank" rel="noopener">
  Website Pemkot
</a>

Menandai Menu Aktif di Navbar

'use client'  // ← wajib jika pakai hook

import { usePathname } from 'next/navigation'

export default function Navbar() {
  const pathname = usePathname()  // Mendapatkan URL saat ini
  
  return (
    <nav>
      <Link
        href="/profil"
        className={pathname === '/profil' ? 'aktif' : ''}
      >
        Profil
      </Link>
    </nav>
  )
}
🔵
Kapan pakai 'use client'?
Tambahkan di bagian atas file jika komponen menggunakan:
• useState — menyimpan data yang bisa berubah
• useEffect — menjalankan kode setelah render
• Event handler seperti onClick, onChange
• Hook Next.js seperti usePathname, useRouter

⚙️ 10. Build menjadi Static Site

1 Tambahkan Config Static Export

Pastikan next.config.js berisi ini:

/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',  // ← Ini yang membuat jadi static
}

module.exports = nextConfig
2 Jalankan Build
npm run build

Jika berhasil, akan muncul folder out/ berisi file HTML statis yang siap upload.

🚀 11. Deploy ke Vercel

1 Upload ke GitHub Dulu

Vercel membaca langsung dari GitHub. Buat akun di github.com jika belum punya.

# Di terminal, di folder project:
git init
git add .
git commit -m "pertama kali upload"

# Buat repository baru di GitHub, lalu:
git remote add origin https://github.com/USERNAME/pondok-pucung.git
git push -u origin main
2 Deploy di Vercel
  1. Buka vercel.com → Sign up with GitHub
  2. Klik tombol "Add New... → Project"
  3. Pilih repository pondok-pucung
  4. Klik "Deploy"
  5. Tunggu 1-2 menit... selesai! 🎉
3 Update Website

Setelah terhubung ke Vercel, setiap kali kamu push ke GitHub, website otomatis update!

# Setelah edit file, jalankan:
git add .
git commit -m "update berita bulan mei"
git push

# Vercel akan auto-deploy dalam ~1 menit
✅
Vercel gratis untuk project personal!
Kamu akan mendapatkan URL seperti: pondok-pucung.vercel.app
Bisa juga menggunakan domain sendiri jika ada.
